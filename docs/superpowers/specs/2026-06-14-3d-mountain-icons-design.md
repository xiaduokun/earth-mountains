# 3D 写实山峰图标设计

**日期**: 2026-06-14
**目标**: 用基于真实照片纹理的 3D 山体模型替换当前扁平的 HTML 照片标记

---

## 概述

当前山峰标记使用 `<Html>` 层中的小尺寸 `<img>` 标签（2-4px），缺乏 3D 体积感。本设计将其替换为具有真实照片纹理的 3D 山体网格（LatheGeometry），在 3D 地球上实现写实、有体积感的展示效果。

## 几何体

- **类型**: `LatheGeometry`（旋转体），生成类似真实山体的轮廓
- **轮廓曲线**: 底部宽 → 中段加速收窄 → 顶部尖峰
- **径向分段**: 16（提供平滑的圆形底座）
- **高度分段**: 8（沿轮廓的足够细节）
- 替代旧的 `ConeGeometry`（6 个径向分段的六边形锥体）和 `createMountainGeometry()`

## 纹理

- 使用 `@react-three/drei` 的 `useTexture` 从 `/public/mountains/<id>.jpg` 加载真实山峰照片
- 作为 `MeshStandardMaterial.map` 应用
- 山峰顶点 UV 映射到几何体顶点，山脚映射到底部
- 保留 7 座山的现有 JPEG 照片（400x267，文森峰为 400x500）

## 尺寸系统

基于真实海拔高度，使用线性映射：

| 山峰 | 真实高度（米） | 3D 高度（单位） |
|---|---|---|
| 珠穆朗玛峰 | 8,848 | 0.12 |
| 阿空加瓜山 | 6,961 | 0.095 |
| 德纳里山 | 6,190 | 0.085 |
| 乞力马扎罗山 | 5,895 | 0.08 |
| 勃朗峰 | 4,808 | 0.07 |
| 文森峰 | 4,892 | 0.07 |
| 库克山 | 3,724 | 0.06 |

公式: `3D高度 = 0.05 + (真实高度 - 3724) / (8848 - 3724) × 0.07`

底部半径: `3D 高度 × 0.4`

## 定位

- 每座山使用 `latLngToPosition()` 定位在地球表面
- 山体径向朝外（法线方向），垂直于球面
- 底部贴合球面，如同从地面生长出来

## 组件改动

### MountainMarker（EarthScene.tsx）

从基于 HTML 的标记改为真正的 3D 网格：

**之前**: `<Html>` + `<img>`（扁平的 DOM 叠加层照片）
**之后**: `<group>` + `<LatheGeometry>` + `<MeshStandardMaterial>`（具有照片纹理的 3D 网格）

### 移除的代码
- `MountainMarker` 中的 `<Html>` 照片层
- `App.css` 中的 `.mountain-photo-marker` 和 `.marker-photo` 样式
- `geo.ts` 中旧的 `createMountainGeometry()`（不再被引用）

## 光照

现有场景光照（环境光 + 方向光）已足以让 `MeshStandardMaterial` 产生立体明暗。不需要额外光源。

## 交互

- **悬停**: 山体缩放至 1.5 倍（3D 网格缩放比 HTML 照片放大更具冲击力）
- **视点淡化**: 使用 `material.opacity` 进行淡入/淡出，代替旧的 DOM 透明度操作
- 背对相机时材质透明，正对相机时恢复不透明
- 点击：保留现有的图例/详情卡片行为

## 性能

- 总共 7 个纹理网格——非常轻量，不会出现性能问题
- 纹理已存在于 `/public/mountains/`（每个 11-35 KB）
- 不需要 mipmap（标记载体太小，无需多级渐远）
- 低端设备上考虑使用 `texture.minFilter = THREE.LinearFilter`
